# Required Pages System

Սա նոր համակարգ է, որը պահանջում է, որ օգտատերերը բոտից օգտվելու համար բաժանորդագրվեն կոնկրետ Telegram էջերին։

## Ադմին հրամաններ

### Էջ ավելացնելու համար
```
/addpage @channel_name
```
- Ավելացնում է նոր էջ (ալիք կամ բոտ)
- Օրինակ՝ `/addpage @domino_updates`

### Էջերի ցուցակը տեսնելու համար
```
/listpage
```
- Ցույց է տալիս բոլոր պահանջվող էջերը ID-ով և username-ով

### Էջ ջնջելու համար
```
/delpage ID
```
- Ջնջում է կոնկրետ էջը ID-ով
- Օրինակ՝ `/delpage 3`

## Ինչպես է աշխատում

1. **Ադմինը** ավելացնում է պահանջվող էջերը `/addpage` հրամանով
2. **Օգտատերը** սեղմում է "Open App" կոճակը
3. **Բոտը** ստուգում է՝ արդյոք օգտատերը բոլոր էջերին բաժանորդագրված է
4. **Եթե այո**՝ բացում է WebApp-ը
5. **Եթե ոչ**՝ ցույց է տալիս հաղորդագրություն՝ բացակայող էջերի ցուցակով

## Տեխնիկական մանրուքներ

- **Բազա**՝ `dom_required_pages` աղյուսակ
- **API**՝ `/api/check_required_pages` endpoint
- **Frontend**՝ `page-check.js` ֆայլ
- **Ստուգում**՝ Telegram Bot API `getChatMember` մեթոդով

## Օգտագործման օրինակ

```
1. /addpage @domino_channel
2. /addpage @domino_bot
3. /listpage
   ID: 1 | @domino_channel | Добавлена: 2025-01-15 10:30
   ID: 2 | @domino_bot | Добавлена: 2025-01-15 10:31
4. /delpage 2
```

## Անվտանգություն

- Հրամանները հասանելի են միայն ադմինիստրատորներին (ADMIN_IDS)
- Telegram API-ն ավտոմատ ստուգում է իրական անդամակցությունը
- Սխալների դեպքում օգտատերը չի ստանում մուտքի իրավունք
