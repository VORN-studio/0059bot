# Telegram Pages Verification System Setup

## Ի՞նչ է արվել

Ավելացվել է Telegram էջերի ստուգման համակարգ, որը թույլ է տալիս՝
- Ադմինին ավելացնել Telegram էջեր
- Ստուգել օգտատերերին՝ արդյոք նրանք ֆոլոու են արել էջերին
- Թույլ չտալ բոտ մտնելը, եթե օգտատերը ֆոլոու չէ

## Տեղադրման քայլեր

### 1. Տեղադրել նոր գրադարանները

```bash
pip install -r requirements.txt
```

### 2. Ստանալ Pyrogram API հավատարման տվյալներ

1. Այցելեք https://my.telegram.org
2. Մուտք գործեք ձեր Telegram հաշվով
3. Գնացեք "API development tools"
4. Ստեղծեք նոր հավելված
5. Ստացեք API ID և API Hash

### 3. Կարգավորել environment variables

Պատճենեք `.env.example` ֆայլը `.env`:

```bash
cp .env.example .env
```

Խմբագրեք `.env` ֆայլը՝ ավելացնելով ձեր API հավատարման տվյալները:

```
PYROGRAM_API_ID=12345678
PYROGRAM_API_HASH=abcdef1234567890abcdef1234567890
```

### 4. Վերամեկնարկել բոտը

```bash
python bot.py
```

## Ադմին հրամաններ

### `/addpage <link>`
Ավելացնում է նոր էջ ստուգման համար։

Օրինակներ:
- `/addpage @mypage`
- `/addpage https://t.me/mypage`

### `/listpage`
Ցուցադրում է բոլոր ավելացված էջերը։

### `/delpage <id>`
Հեռացնում է էջը ID-ով։

Օրինակ:
- `/delpage 1`

## Ինչպես է աշխատում

1. **Ադմինը** ավելացնում է էջեր `/addpage` հրամանով
2. **Օգտատերը** սեղմում է `/start`
3. **Բոտը** ստուգում է՝ արդյոք օգտատերը ֆոլոու է բոլոր էջերին
4. **Եթե այո**՝ բոտը բացվում է
5. **Եթե ոչ**՝ բոտը ուղարկում է հաղորդագրություն՝ էջերի ցուցակով

## Նշումներ

- Բոտը պետք է ադմին լինի այն էջերում, որոնք ստուգում է
- Եթե Pyrogram client-ը չի կարողանում միանալ, բոտը կթույլ տա մուտքը առանց ստուգման
- Էջերի տվյալները պահվում են `telegram_pages` աղյուսակում
